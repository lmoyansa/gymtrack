# Sprint 0 - Definición inicial y preparación de la gestión

## Periodo

**Fecha:** 02/07/2026

## Objetivo del sprint

El objetivo del Sprint 0 fue definir inicialmente el proyecto GymTrack y preparar el sistema de gestión que se utilizaría durante los siguientes sprints.

Este sprint no tenía como finalidad producir una versión funcional de la aplicación, sino establecer una base mínima para organizar el trabajo posterior.

## Trabajo realizado

Durante el Sprint 0 se completaron las siguientes tareas:

- Definición inicial de las historias de usuario.
- Creación del tablero de gestión en Notion.

Como parte de estas tareas también se realizó una primera delimitación del producto mínimo viable y se configuraron las propiedades necesarias para clasificar el trabajo:

- Sprint.
- Historia de usuario relacionada.
- Estado.
- Responsable simulado.
- Tipo de tarea.
- Prioridad.
- Descripción.
- Puntos de historia.

## Entregables generados

- Primera versión de las historias de usuario.
- Delimitación inicial del MVP.
- Tablero de tareas en Notion.
- Organización inicial del proyecto por sprints.
- Definición de los perfiles simulados del proyecto.

## Puntos de historia

El Sprint 0 alcanzó un total de **6 puntos de historia**.

## Estado del sprint

**Sprint 0 finalizado.**

Las tareas previstas fueron completadas y se preparó la base necesaria para comenzar el análisis detallado del proyecto.

El Sprint 0 permitió comenzar el proyecto con una definición básica del producto y una herramienta de seguimiento. A partir de esta base, el Sprint 1 se dedicaría a refinar los requisitos, preparar el prototipo, definir el modelo de datos y organizar el repositorio y las pruebas.
