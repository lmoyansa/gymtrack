## Fecha de inicio

## Fecha de revisión

## Objetivo del sprint

Preparar la base del proyecto antes de comenzar la implementación principal de la aplicación. Este sprint se centra en análisis, prototipo, repositorio, estructura inicial, modelo de datos y estrategia de pruebas.

## Tareas principales

- Revisar historias de usuario.
- Definir tareas y subtareas por historia de usuario.
- Crear prototipo inicial de pantallas.
- Definir flujo de navegación.
- Definir modelo de datos inicial.
- Documentar atributos y relaciones.
- Definir estrategia de pruebas.
- Crear repositorio en GitHub.
- Definir política de ramas.
- Definir política de commits.
- Crear README inicial.
- Crear carpeta de documentación.
- Preparar documento de gestión del repositorio.

## Resultado esperado

Al finalizar el Sprint 1 se espera disponer de:

- Prototipo inicial de la aplicación.
- Flujo de navegación documentado.
- Modelo de datos inicial.
- Estrategia de pruebas definida.
- Repositorio GitHub creado.
- README inicial.
- Documentación básica del proyecto.
- Tablero de Notion actualizado con tareas, subtareas y sprints.
