# Reunión 01 - Seguimiento y planificación del Sprint 1

**Fecha:** Lunes, 06/07/2026  
**Hora:** 11:00  
**Asistentes:** Luis Moyano y Baldomero Imbernón Tudela  
**Sprint:** Sprint 1

## Objetivo de la reunión

Revisar el enfoque inicial del proyecto GymTrack y concretar los elementos que debían prepararse durante el Sprint 1 antes de comenzar la implementación principal.

## Temas tratados

- Revisión de la propuesta inicial de GymTrack.
- Organización del proyecto mediante una metodología ágil adaptada.
- Necesidad de incorporar el perfil de analista.
- Preparación de un prototipo antes de comenzar la implementación.
- Definición de tareas y subtareas por historia de usuario.
- Organización del trabajo mediante Sprint 0, Sprint 1 y Sprint 2.
- Definición inicial de pruebas unitarias, de integración y de aceptación.
- Utilización de GitHub para el control de versiones.
- Utilización de Notion para gestionar el backlog y los sprints.
- Necesidad de preparar el modelo de datos y el flujo de navegación.

## Decisiones tomadas

- Mantener Scrum como metodología de referencia, adaptada al desarrollo individual.
- Añadir el perfil simulado de analista.
- Organizar el proyecto mediante un Sprint 0 y dos sprints principales.
- Añadir la propiedad Sprint al tablero de Notion.
- Descomponer las historias de usuario en tareas y subtareas.
- Definir criterios de aceptación para cada historia.
- Crear un prototipo inicial mediante Figma.
- Definir el flujo de navegación antes de programar.
- Preparar el modelo de datos inicial.
- Definir la estrategia de pruebas antes de completar la implementación.
- Crear y organizar el repositorio de GitHub.

## Tareas pendientes

- Revisar las historias de usuario.
- Definir tareas y subtareas.
- Definir criterios de aceptación.
- Crear el prototipo.
- Preparar el flujo de navegación.
- Definir el modelo de datos.
- Definir la estrategia de pruebas.
- Crear el repositorio de GitHub.
- Definir la política de ramas y commits.
- Crear la carpeta de documentación.
- Preparar el documento de gestión del repositorio.

## Resultado

La reunión permitió concretar el contenido del Sprint 1 y aclarar que su finalidad sería reducir la incertidumbre antes de comenzar la implementación principal.
