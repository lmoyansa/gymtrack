# Reunión 02 - Revisión del Sprint 1 y planificación del Sprint 2

**Fecha:** Viernes, 10/07/2026  
**Hora:** 10:30  
**Asistentes:** Luis Moyano y Baldomero Imbernón Tudela  
**Sprint:** Cierre del Sprint 1 e inicio del Sprint 2

## Objetivo de la reunión

Revisar los resultados obtenidos durante el Sprint 1 y definir el orden de trabajo para implementar y validar el MVP durante el Sprint 2.

## Elementos revisados

- Historias de usuario.
- Tareas y subtareas.
- Criterios de aceptación.
- Prototipo inicial.
- Flujo de navegación.
- Modelo de datos.
- Estrategia de pruebas.
- Repositorio de GitHub.
- Política de ramas.
- Política de commits.
- Proyecto inicial de Android Studio.
- Documentación generada.
- Estado del tablero de Notion.

## Resultado de la revisión

Se comprobó que las tareas previstas para el Sprint 1 habían sido completadas.

El prototipo, el modelo de datos, los criterios de aceptación y la estrategia de pruebas proporcionaban una base suficiente para comenzar el desarrollo funcional de la aplicación.

## Aspectos detectados

- Algunas tareas de análisis se habían confundido inicialmente con tareas de implementación.
- El modelo de datos había necesitado varias revisiones.
- La preparación de documentación había requerido más tiempo del previsto.
- Todavía no existía una funcionalidad completa y ejecutable.

## Decisiones tomadas

- Considerar finalizado el Sprint 1.
- Comenzar el Sprint 2 con la creación de las clases y entidades principales.
- Configurar Room después de disponer del modelo de datos.
- Priorizar la gestión de rutinas y ejercicios.
- Implementar posteriormente los entrenamientos y el historial.
- Incorporar el cronómetro al flujo del entrenamiento.
- Ejecutar pruebas después de cada bloque importante.
- Mantener actualizado el tablero de Notion.
- Utilizar ramas `feature` para separar los cambios.
- Corregir los errores bloqueantes antes de continuar con nuevas funcionalidades.

## Tareas previstas para el Sprint 2

- Crear las clases del modelo.
- Configurar Room y los DAO.
- Implementar registro, login y cierre de sesión.
- Implementar la pantalla principal.
- Implementar la gestión de rutinas.
- Implementar la gestión de ejercicios.
- Implementar el registro de entrenamientos.
- Implementar el historial.
- Implementar el cronómetro.
- Ejecutar las pruebas unitarias.
- Ejecutar las pruebas de integración.
- Ejecutar las pruebas de aceptación.
- Corregir las incidencias.
- Preparar una versión estable.

## Resultado final

El Sprint 1 quedó cerrado y se aprobó el comienzo de la implementación y validación del MVP durante el Sprint 2.
